## How to Write Code Here

- Read before you write. Understand existing code before modifying it.
- One change at a time. Make it, test it, move on.
- Fix root causes. Don't layer patches on symptoms.
- Match the codebase. Use the patterns already established here.

## Prohibited Patterns

- Don't add error handling for impossible cases.
- Don't add comments that restate the code.
- Don't create abstractions for single use cases.
- Don't refactor code you weren't asked to touch.
- Don't add dead code "for later."
