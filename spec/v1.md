# V1

## What Ships

A TUI dashboard that:
- Spawns multiple AI CLI agents (claude, codex, etc.)
- Displays them in a configurable grid layout
- Lets the user switch between agents and type directly into them
- Shows agent status (working/idle) at a glance

## What Does NOT Ship

- Task queue / task management
- Agent-to-agent communication
- Performance tracking / incentivization
- Session persistence / resume
- File tree panel
- Hot-reload of layout

## Primitives

- **Agent** — Process in a PTY with terminal emulation
- **Widget** — Stateless render component
- **Grid** — 12-column layout from `layout.json`

## Default Layout

```
┌────────────────────┬─────────────────────────────────────────────┐
│ agent-list (3 col) │ agent-pane (9 col)                         │
│                    │                                             │
│                    │                                             │
│                    │                                    80% height│
├────────────────────┴─────────────────────────────────────────────┤
│ status-bar (12 col)                                    20% height│
└──────────────────────────────────────────────────────────────────┘
```

## Keybindings

| Key | Action |
|-----|--------|
| ctrl+n | Spawn new agent |
| ctrl+w | Kill focused agent |
| ctrl+j | Focus next agent |
| ctrl+k | Focus previous agent |
| ctrl+q | Quit |

Everything else is forwarded to the focused agent's stdin.
